# 429 重试次数配置方案

## 背景

当前最大重试次数为硬编码常量 `MAX_TOTAL_RETRIES = 4`（[provider.rs:31](../src/kiro/provider.rs#L31)）。
每次 429 重试使用长退避（base 1s，封顶 8s），4次重试合计等待约 **7~9s** 后才向用户返回错误。

目标：将上限调整为 9 次，并可选地支持前端运行时配置。

---

## 方案 A：直接改常量（推荐先验证）

**改动范围：1 个文件，1 行**

[src/kiro/provider.rs:31](../src/kiro/provider.rs#L31)：

```rust
// 改前
const MAX_TOTAL_RETRIES: usize = 4;

// 改后
const MAX_TOTAL_RETRIES: usize = 9;
```

**注意**：同时更新第 429 行的注释（「硬上限 9 次」），使注释与实际值一致。

**优点**：零风险，一行改完。  
**缺点**：每次调整都需要重新编译部署。

---

## 方案 B：挂到 throttle config 接口，前端可配置

完全复用 `account_throttle_cooldown_secs` 的现有模式，无需新建接口或新组件。

### 改动清单

#### 1. `src/model/config.rs`

在 `KiroConfig` 结构体中加字段，紧挨 `account_throttle_cooldown_secs`：

```rust
/// 最大总重试次数（默认 9）
#[serde(default = "default_max_total_retries")]
pub max_total_retries: usize,
```

加 default fn 和 Default impl 填充：

```rust
fn default_max_total_retries() -> usize { 9 }
```

#### 2. `src/kiro/token_manager.rs`

在 `TokenManager` 结构体加字段（紧挨 `account_throttle_cooldown_secs`）：

```rust
max_total_retries: AtomicUsize,
```

初始化（`TokenManager::new` 中）：

```rust
max_total_retries: AtomicUsize::new(config.max_total_retries),
```

加 getter：

```rust
pub fn get_max_total_retries(&self) -> usize {
    self.max_total_retries.load(Ordering::Relaxed)
}
```

扩展 `set_account_throttle_config`，接受新参数并写入：

```rust
pub fn set_account_throttle_config(
    &self,
    failover: Option<bool>,
    cooldown_secs: Option<u64>,
    max_total_retries: Option<usize>,   // 新增
) -> anyhow::Result<()> {
    // 校验
    if let Some(n) = max_total_retries {
        if !(1..=20).contains(&n) {
            anyhow::bail!("max_total_retries 必须在 1..=20: {}", n);
        }
    }
    // ... 原有逻辑不变 ...
    if let Some(n) = max_total_retries {
        self.max_total_retries.store(n, Ordering::Relaxed);
    }
    Ok(())
}
```

#### 3. `src/kiro/provider.rs`

删除常量 `MAX_TOTAL_RETRIES`，两处使用改为读 token_manager：

```rust
// call_api_with_retry（约第 440 行）
let max_retries = (total_credentials * MAX_RETRIES_PER_CREDENTIAL)
    .min(self.token_manager.get_max_total_retries());

// call_mcp_with_retry（约第 270 行）
let max_retries = (total_credentials * MAX_RETRIES_PER_CREDENTIAL)
    .min(self.token_manager.get_max_total_retries());
```

#### 4. `src/admin/types.rs`

`AccountThrottleConfigResponse` 和 `SetAccountThrottleConfigRequest` 各加一个字段：

```rust
pub struct AccountThrottleConfigResponse {
    pub failover: bool,
    pub cooldown_secs: u64,
    pub max_total_retries: usize,   // 新增
}

pub struct SetAccountThrottleConfigRequest {
    pub failover: Option<bool>,
    pub cooldown_secs: Option<u64>,
    pub max_total_retries: Option<usize>,   // 新增
}
```

#### 5. `src/admin/service.rs`

`get_account_throttle_config` 中填充新字段，`set_account_throttle_config` 透传参数：

```rust
// get
max_total_retries: self.token_manager.get_max_total_retries(),

// set
self.token_manager.set_account_throttle_config(
    req.failover,
    req.cooldown_secs,
    req.max_total_retries,   // 新增
)
```

#### 6. `admin-ui/src/types/api.ts`

```ts
export interface AccountThrottleConfig {
  failover: boolean;
  cooldownSecs: number;
  maxTotalRetries: number;   // 新增
}
```

#### 7. `admin-ui/src`（风控配置卡片组件）

在已有的冷却时长输入框旁边，加一个数字输入框：

```tsx
<label>最大重试次数（1–20）</label>
<input
  type="number" min={1} max={20}
  value={config.maxTotalRetries}
  onChange={e => setMaxTotalRetries(Number(e.target.value))}
/>
```

### 改动汇总

| 文件 | 性质 | 估计行数 |
|------|------|---------|
| config.rs | 加字段 + default fn | ~6 |
| token_manager.rs | 加字段 + getter + 扩展 setter | ~15 |

共 **7 个文件**，每处均为填空式改动，无新接口、无新组件。

---

## 选择建议

1. 先用**方案 A**（改常量为 9）上线验证，确认效果。
2. 若后续需要频繁调整，再按**方案 B**做前端配置，代价可控。
